# 클립 1-4 결과물 예시 · 경쟁사 신제품 비교표 자동 생성

> Firecrawl MCP 설치 직후 마케터가 받게 되는 첫 결과물. `firecrawl-mcp` (npm v3.17.0 · firecrawl.dev 공식) 기반.

## 한 줄 요약

사용자가 경쟁사 도메인 3개 + 비교 항목 지정 → Claude 가 각 사이트 scrape + extract → 5분 안에 비교표 + 인사이트 마크다운 응답.

---

## 🎬 사용자 입력 (예시)

```
다음 3개 사이트의 신제품 정보를 비교표로 만들어줘:
- competitor-a.com/products/new
- competitor-b.com/launches
- competitor-c.com/spring-2026

비교 항목: 제품명, 가격, 핵심 기능 3개, 출시일, 차별점
```

## 📊 결과물 1 · 경쟁사 신제품 비교표

```
✅ 3개 사이트 스크랩 완료 (소요 1분 20초)

| 항목 | Competitor A | Competitor B | Competitor C |
|---|---|---|---|
| 제품명 | Hydra Glow Serum | Aqua Boost Essence | Moisture Lock Toner |
| 가격 | ₩42,000 | ₩38,000 | ₩29,000 |
| 핵심 기능 1 | 5% 나이아신아마이드 | 히알루론산 5중복합 | 세라마이드 + AHA |
| 핵심 기능 2 | 비건 인증 | 동물실험 X | 저자극 EWG 그린 |
| 핵심 기능 3 | 무향료 | 라벤더 향 | 천연 식물성 향 |
| 출시일 | 2026-05-10 | 2026-05-15 | 2026-05-20 |
| 차별점 | 임상 4주 효과 데이터 공개 | 한정판 패키지 | 가격 경쟁력 (-23%) |

📌 인사이트 (Claude 자동 도출)
  · 가격 포지셔닝: C 가격이 가장 낮음 (₩29K), A 프리미엄 (₩42K)
  · 차별 전략: A 는 효과 입증, B 는 디자인, C 는 가격
  · 본인 브랜드 대응: 가격은 B 와 동일선, 효과 입증 데이터 보강 시 A 와 경쟁 가능

🔗 원본 소스
  · competitor-a.com/products/new (스크랩 시간 24초)
  · competitor-b.com/launches (스크랩 시간 31초)
  · competitor-c.com/spring-2026 (스크랩 시간 28초)

📊 크레딧 사용: 18 / 500 (월 무료 할당량)
```

## 🧠 마케터 활용법 6가지

### A. 경쟁사 모니터링 (가장 자주 씀)
```
"경쟁사 5개 도메인 신규 페이지·블로그·프로모션 변경사항 매주 자동 추적"
→ Part 4 competitor-monitor 에이전트 결합
```

### B. 트렌드 리서치
```
"패션 매거진 5개 사이트에서 이번 시즌 트렌드 키워드 추출"
→ scrape + extract 도구로 헤드라인·태그·인기 글 수집
```

### C. 콘텐츠 인풋 자동화
```
"이 URL 5개 본문을 마크다운으로 추출해줘 — 뉴스레터 소스로"
→ scrape 도구 5회 · Part 3 newsletter-writer 에이전트로 전달
```

### D. 가격 모니터링
```
"competitor-a.com/products 전체 페이지의 가격 추출 후 어제 대비 변화"
→ crawl 도구 + 일자별 .env 비교
```

### E. 백링크/SEO 페이지 분석
```
"네이버 블로그 'XX' 검색 결과 상위 10개 URL 추출 + 제목·요약"
→ search 도구 활용
```

### F. 리뷰 텍스트 마이닝
```
"본인 브랜드 + 경쟁사 3개 네이버 리뷰 페이지 본문 크롤링"
→ crawl + Part 4 review-analyzer 에이전트로 전달
```

## 🎯 본인 데이터로 따라하기 (5분)

1. `firecrawl.dev` 가입 → API key 발급
2. `.env` 에 `FIRECRAWL_API_KEY=fc-xxx` 등록
3. Claude Code 에서:
   ```
   다음 2개 사이트의 신제품 비교표 만들어줘:
   - <본인 경쟁사 1 URL>
   - <본인 경쟁사 2 URL>
   ```
4. 1~2분 대기 → 비교표 + 인사이트 응답
5. "본인 브랜드 대응안 추가해줘" 후속 질의

## 💡 자주 묻는 질문

**Q. 어떤 사이트도 다 스크랩 되나요?**  
A. 대부분 OK. 단, robots.txt 차단 / Cloudflare 강력 보호 / 로그인 필수 사이트는 어려움. SPA(React 등) 도 대응 됨 (헤드리스 브라우저 사용).

**Q. 무료 500크레딧 어떻게 계산되나요?**  
A. 1 URL scrape = 1 크레딧. crawl 은 페이지 수만큼. 강의 학습 + 본인 모니터링은 충분.

**Q. JavaScript 가 렌더링되는 사이트도 가능한가요?**  
A. 네. Firecrawl 클라우드 헤드리스 브라우저가 JS 실행 후 결과 반환.

**Q. 한국어 사이트도 잘 되나요?**  
A. 네. UTF-8 출력. 네이버 블로그·쿠팡·다음 등 모두 검증됨.

**Q. 동시에 사이트 10개 스크랩 가능한가요?**  
A. `batch_scrape` 도구 사용 (한 번에 10개 URL). 무료 플랜은 분당 10회 제한.
