# 클립 1-4 실습 · Firecrawl MCP · 쿠팡 리뷰 크롤링 + 경쟁사 리뷰 분석

## 목표 산출물

1. Firecrawl(파이어크롤) API 키 발급 + 등록 (초기 1000 크레딧)
2. **본인 제품 쿠팡 리뷰** 마크다운으로 추출
3. **경쟁사 제품 쿠팡 리뷰** 1~2개 추출 + 비교 분석 표

## 왜 쿠팡 리뷰인가

- 기존 시도 : 쿠팡은 JavaScript 동적 렌더링 + 강한 봇 차단으로 일반 크롤러로는 리뷰 본문 추출이 거의 불가
- Firecrawl 해결 : 헤드리스 브라우저 + JS 렌더 대기(`waitFor`) + 모바일 User-Agent 옵션으로 안정적 추출
- 결과 : 본인 제품 + 경쟁사 제품의 리뷰 본문·별점·작성일을 한 줄 명령으로 수집 → Claude(클로드) 가 긍·부정 키워드, 강점·약점, 마케팅 카피 인사이트까지 자동 정리

## 단계별 명령

### Step 1. API 키 발급 (1000 크레딧 무료)

```
1. https://firecrawl.dev 가입 (Google 로그인 가능)
2. Dashboard → API Keys → Create API Key
3. 키 1회만 표시 → 즉시 복사
4. 초기 1000 크레딧 자동 지급 (가입 직후 확인)
   · 1 페이지 scrape = 1 크레딧
   · 본 실습 전체 소진량 약 10~20 크레딧
```

> 💡 1000 크레딧은 본 실습 50회 이상 반복 가능. 본인 카테고리 경쟁사 10곳을 매주 점검해도 한 달은 무료.

### Step 2. .env 등록

```bash
FIRECRAWL_API_KEY=fc-xxxxxxxxxxxx
```

### Step 3. .mcp.json 확인

```json
{
  "mcpServers": {
    "firecrawl": {
      "command": "npx",
      "args": ["-y", "firecrawl-mcp"],
      "env": {
        "FIRECRAWL_API_KEY": "${FIRECRAWL_API_KEY}"
      }
    }
  }
}
```

### Step 4. 헬스 체크

```
claude
> /mcp     # firecrawl ✅ 확인
> "https://example.com 페이지를 마크다운으로 가져와줘"
# → 본문 마크다운 응답 시 정상
```

## 시연 1 · 본인 제품 쿠팡 리뷰 추출

```
[역할] 너는 쿠팡 셀러 카테고리 분석가야.

[입력] 쿠팡 상품 URL 1개:
  https://www.coupang.com/vp/products/{본인_상품ID}

[수행]
  1. firecrawl_scrape 호출 · 옵션: formats=['markdown'], waitFor=3000, mobile=true
  2. 페이지 본문에서 다음 추출
     · 상품명 · 가격 · 별점 평균
     · 최신 리뷰 20건 (별점 · 작성일 · 본문)
  3. 결과를 마크다운 표로 정리

[제약] 쿠팡 봇 차단 발생 시 waitFor 를 5000 으로 올리고 재시도
```

### 예상 산출물

```markdown
## 본인 제품 · 베스트 크림 200ml

| 별점 평균 | 리뷰 수 | 최신 리뷰 |
|---|---|---|
| 4.6 | 1,247 | 20건 추출 |

| 별점 | 작성일 | 본문 (요약) |
|---|---|---|
| ★★★★★ | 2026-05-22 | "흡수 빠르고 끈적임 없음" |
| ★★★ | 2026-05-21 | "용량 대비 가격 아쉬움" |
| ... | ... | ... |
```

## 시연 2 · 경쟁사 제품 리뷰 비교 분석

```
[역할] 너는 K뷰티 D2C 카테고리 경쟁 분석가야.

[입력] 비교 대상 쿠팡 상품 3개:
  · 본인  : https://www.coupang.com/vp/products/{본인_ID}
  · 경쟁A : https://www.coupang.com/vp/products/{경쟁A_ID}
  · 경쟁B : https://www.coupang.com/vp/products/{경쟁B_ID}

[수행]
  1. 3개 URL 각각 firecrawl_scrape · 최신 리뷰 20건씩 (총 60건)
  2. 각 제품별 긍정 키워드 TOP 5 · 부정 키워드 TOP 5 추출
  3. 강점·약점 비교 표 작성
  4. 본인 제품 광고 카피 개선 포인트 3가지 도출

[산출물] 마크다운 표 3개 :
  · 표 1 : 제품별 별점·리뷰 수
  · 표 2 : 긍정·부정 키워드 비교
  · 표 3 : 마케팅 카피 개선 포인트

[제약] 한국어. 별점 1~2점 리뷰는 부정 카테고리로 자동 분류
```

### 예상 산출물 · 표 2 (긍정·부정 키워드 비교)

| 제품 | 긍정 TOP 3 | 부정 TOP 3 |
|---|---|---|
| 본인 | 흡수·향·용기 디자인 | 가격·용량·끈적임 |
| 경쟁A | 가격·향·용량 | 흡수 느림·향 강함 |
| 경쟁B | 발림성·고급감 | 가격·소량 |

→ 본인 제품의 차별점 : 흡수력 · 약점 : 가격 부담
→ 광고 카피 개선 : "끈적임 없는 빠른 흡수" 강조 · "동급 대비 가격 정당화 메시지" 추가

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| 쿠팡 페이지 빈 응답 | `waitFor=5000, mobile=true` 명시. JS 렌더 시간 확보 |
| `403 Forbidden` (봇 차단) | `mobile=true` 또는 다른 User-Agent. 그래도 안 되면 60초 대기 후 재시도 |
| Rate limit | `claude mcp restart firecrawl` 후 60초 대기 |
| 한글 깨짐 | `formats=['markdown']` 명시 |
| 리뷰 일부만 추출 | 쿠팡 페이지네이션 별도. `waitFor` 늘리고 스크롤 옵션 안내문에 따라 재시도 |
| 크레딧 빠른 소진 | scrape 1회 = 1 크레딧. crawl 은 페이지 수만큼 차감. 도메인 전체 크롤 자제 |
