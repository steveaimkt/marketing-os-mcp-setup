# Part 2 / 1-4 · Firecrawl 실습 — 쿠팡 리뷰 분석 자동화

> 쿠팡 상품 URL 만 던지면 Claude 가 리뷰를 긁어와 긍·부정 키워드로 분류하고 광고 카피 개선 포인트까지 도출한다.
> Firecrawl MCP 의 `scrape` + 옵션 조합을 한 흐름에 시연하는 클립 1-4 실습 스킬.

## 전제 조건

- Firecrawl MCP 연결됨 (`mcp__firecrawl__*` 도구 노출). 안 보이면 `marketing-os` 루트에서 Claude Code 재시작.
- `.env` 에 `FIRECRAWL_API_KEY` 등록됨 (firecrawl.dev 가입 시 1000 크레딧 자동 지급).
- 분석할 쿠팡 상품 URL 1개 이상 (본인 제품). 경쟁사 URL 0~2개는 시연 2 단계에서 옵션.

## 진행 포맷 (4단계 — 끊지 말고 한 흐름으로)

### 1️⃣ 안내 인사

다음 멘트로 시작:

```
🔥 Firecrawl 실습을 시작합니다.
기존 크롤러로 안 되던 쿠팡 리뷰 분석을 한 줄 명령으로 자동화합니다.
본인 제품 리뷰 추출(시연 1) → 경쟁사 비교 분석(시연 2) 두 단계로 진행해요.
```

### 2️⃣ 실습 가능한 업무 (대표 5개 표로 안내)

| # | 업무 | 도구 |
|---|---|---|
| 1 | **본인 쿠팡 상품 리뷰 추출 (20건)** ⭐ | `firecrawl_scrape` |
| 2 | **본인 + 경쟁사 2개 리뷰 비교 분석 (60건)** ⭐ | `firecrawl_scrape` × 3 |
| 3 | 네이버 쇼핑·11번가 리뷰 추출 | `firecrawl_scrape` |
| 4 | 경쟁사 D2C 사이트 신제품 가격·USP 표 | `firecrawl_extract` |
| 5 | 매주 자동 모니터링 (Part 4 연결) | `firecrawl_monitor_create` + cron |

이어서 묻기:
```
1번 본인 제품 리뷰부터 시작할까요? 쿠팡 상품 URL 1개를 알려주세요.
(경쟁사 비교까지 한 번에 가려면 URL 2~3개 한꺼번에 주셔도 됩니다)
```

### 3️⃣ 결과물 만들기 (시연 1 + 시연 2 자동 실행)

#### 시연 1 · 본인 제품 리뷰 추출

사용자가 URL 1개를 주면 즉시 실행:

```
도구: firecrawl_scrape
url: <사용자 입력 URL>
formats: ['markdown']
waitFor: 5000
mobile: true
```

받아온 마크다운에서 Claude 가 다음을 직접 추출 :
- 상품명 · 가격 · 별점 평균 · 총 리뷰 수
- 최신 리뷰 20건 (별점 · 작성일 · 본문 요약 1~2줄)

출력 형식 (마크다운 표 2개) :

```markdown
## 시연 1 · 본인 제품 리뷰

| 상품명 | 가격 | 별점 평균 | 리뷰 수 |
|---|---|---|---|
| ... | ... | ... | ... |

| 별점 | 작성일 | 본문 (요약) |
|---|---|---|
| ★★★★★ | 2026-05-22 | ... |
| ... | ... | ... |
```

#### 시연 2 · 경쟁사 리뷰 비교 (URL 2~3개 시)

사용자가 URL 을 2개 이상 줬으면 자동으로 진행. 1개만 줬으면 묻기:
```
경쟁사 쿠팡 URL 1~2개 더 주시면 60건 비교 분석으로 확장합니다. 건너뛰려면 "넘어가" 라고 답해주세요.
```

URL 3개 모두에 `firecrawl_scrape` 호출 → 각 20건씩, 총 60건 리뷰 수집.

Claude 가 다음 4개 표를 자동 생성 :

**표 1 · 제품별 별점·리뷰 수 요약**
```markdown
| 제품 | 별점 평균 | 리뷰 수 | 최신 추출 |
|---|---|---|---|
| 본인 | 4.6 | 1,247 | 20건 |
| 경쟁사 A | 4.4 | 892 | 20건 |
| 경쟁사 B | 4.7 | 2,103 | 20건 |
```

**표 2 · 긍정·부정 키워드 비교 (제품별 TOP 5)**

본문에서 빈도 기반 키워드 추출 + 별점 1~2점은 부정 카테고리로 자동 분류.

**표 3 · 강점·약점 도식**

각 제품의 강점·약점을 한 줄씩.

**표 4 · 본인 제품 광고 카피 개선 포인트 3가지**

본인 제품의 부정 키워드 + 경쟁사 강점에서 도출.

### 4️⃣ 추가 가능 업무 안내 (마무리)

```
✅ 시연 완료. 이 결과로 다음 3가지를 추가로 할 수 있어요.

A. **Google Sheets 1page 리포트로 자동 적재**
   → "이 결과 1page 시트 보고서 양식으로 만들어" 한 줄.
   → 클립 1-2 결과물 양식 그대로 (제목·KPI·비교 표 4종·액션 박스).

B. **매주 자동 모니터링 등록 (Part 4 연결)**
   → competitor-monitor 에이전트가 매주 09시에 같은 URL 재실행
   → 신규 리뷰·별점 변동만 Discord #marketing 채널 발송
   → 본인 명령 0회 · 영구 자동

C. **그 외 사이트로 확장**
   → "네이버 쇼핑 / 11번가 / 다나와 / 블로그 후기" 동일 명령으로 작동.
   → 도구는 firecrawl_scrape 그대로, URL 만 교체.
```

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| 쿠팡 빈 응답 (페이지 본문 없음) | `waitFor` 를 5000 → 8000 으로 올리고 재호출 |
| `403 Forbidden` | `mobile: true` 명시. 그래도 안 되면 60초 대기 |
| 리뷰가 5~10건만 추출됨 | 쿠팡 페이지네이션 한계. 30건+ 필요 시 리뷰 전용 페이지 URL 별도 요청 |
| Rate limit | `claude mcp restart firecrawl` 후 60초 대기 |
| 크레딧 빠른 소진 | scrape 1회 = 1 크레딧. 본 시연 전체 약 3~10 크레딧 사용. 1000 크레딧 무료 분은 한 달 이상 충분 |

## 호출되는 도구

| 도구 | 역할 |
|---|---|
| `firecrawl_scrape` | 쿠팡 페이지 1개 → 마크다운 (JS 렌더 대기 + 모바일 UA) |
| `firecrawl_monitor_create` | (옵션) 매주 자동 재실행 등록 |
| Google Sheets MCP `create_spreadsheet` + `format_cells` | (옵션 A) 1page 시트 적재 |
| Part 4 `competitor-monitor` 에이전트 | (옵션 B) 매주 자동 + Discord 알림 |
