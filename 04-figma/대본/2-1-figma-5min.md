### 🎬 [0:25~1:30] Before/After · PPT 의뢰서 vs Figma 공동작업

〔슬라이드: 02 PPT 의뢰서 vs Figma 공동작업 비교〕

**🎙️ 강사 발화:**

> "지금까지 마케터가 디자이너에게 디자인을 의뢰하는 흐름을 짚어보겠습니다.
>
> **Before(전) · PPT 의뢰서 패턴** · 마케터가 PPT 8장에 와이어프레임 스케치를 그리고, 카피를 정리하고, 흐름을 설명합니다. 약 1~2시간. PPT 를 디자이너에게 메일로 보냅니다. 디자이너는 PPT 를 받아서 Figma 에 다시 그립니다. 약 1시간 추가. 그 후 마케터에게 확인 메일. 마케터가 캡처에 코멘트 달아 회신. 디자이너가 수정. 다시 보내옴. 이 사이클이 반나절~하루 걸립니다.
>
> 여기서 잠깐 짚을 게 있어요. Figma 가 공식 운영하는 호스팅 MCP (mcp.figma.com) 가 있긴 합니다. 그런데 Figma **무료 요금제나 보기 전용 계정**은 **한 달에 6번만 사용 가능**합니다. 실무 자동화엔 턱없이 부족합니다.
>
> 그래서 본 강의는 **자체호스팅 패키지** Talk To Figma MCP Plugin 를 씁니다. WebSocket(웹소켓) 으로 Figma Desktop 앱을 Claude 가 직접 제어. **사용 횟수 제한 없음. 무제한 호출.**
>
> **After(후) · Figma 공동작업 패턴** · 마케터가 Claude 에 자연어 한 줄. **'7가지 마케팅 팁 카드뉴스 8장 와이어프레임 + 카피로 만들어줘.'** 약 2분에 Figma 파일에 와이어프레임 8장이 그려집니다. 디자이너에게 이 Figma 파일 공유 권한만 주면 끝. 디자이너는 **PPT 를 Figma 로 재작성하는 1시간이 사라집니다.** 같은 파일에서 비주얼 폴리시 시작. 마케터는 Figma 코멘트로 실시간 피드백.
>
> **기획안 작성 1~2시간 + 재작성 1시간 + 피드백 반나절 = 거의 하루가 5분 + 실시간 협업으로** 바뀝니다."

### 🎬 [1:30~3:00] 5 시나리오 · 디자이너 의뢰 자료 5가지 자동화

〔슬라이드: 03 5 시나리오 카드〕

**🎙️ 강사 발화:**

> "talk-to-figma 로 자동화되는 마케터 작업 5가지입니다. 본 강의 풀 시연 영상에서는 A 한 가지를 깊이 진행합니다. 모두 **디자이너에게 전달할 의뢰 자료** 라는 점이 핵심입니다.
>
> **A. 원하는 디자인 카드뉴스 8장 ★** · 4단계 흐름. ① 본인이 좋아하는 레퍼런스 카드뉴스를 `scan_text_nodes` + `get_styles` 로 분석 ② 분석 결과를 디자인 시스템 (컬러·폰트·간격 토큰) 으로 구축 ③ Claude 가 8장 콘텐츠 기획 ④ 디자인 시스템 적용한 표지 1장 + `clone_node` × 7 + 텍스트 교체. 총 약 3분. **본 강의 시연.** 이 8장을 디자이너에게 그대로 공유 → 같은 파일에서 협업 시작.
>
> **B. 캠페인 카피 30패턴 일괄 변형** · 본인 배너 1장 헤드라인 30가지 톤 변형. 디자이너에게 '이 중에 어떤 톤이 맞을까요' 물어보기 좋음. 약 30초.
>
> **C. 로그인·랜딩 페이지 와이어프레임** · 모바일·PC 2종. UX 흐름 확정용. 약 2분.
>
> **D. 본인 파일 디자인 시스템 가이드 추출** · 헤드라인·컬러·폰트 마크다운 표로. 디자이너와 톤 합의 시 참고 자료. 약 30초.
>
> **E. 자산 일괄 export** · 페이지의 모든 프레임 webp/svg/png 추출. 의뢰서 첨부용 미리보기 이미지 자동. 약 30초.
>
> 도구는 약 60개. 외울 필요 없어요. Claude 가 알아서 골라 씁니다."

〔→ 04 작동 원리〕

### 🎬 [3:00~4:00] 작동 원리 · 기획 → 와이어프레임 → 디자이너 공동작업

〔슬라이드: 04 작동 흐름 · "figma 시작하자" → 라우터 → 메뉴 → Figma 와이어프레임 → Share → 협업〕

**🎙️ 강사 발화:**

> "작동 원리를 짚어드릴게요.
>
> 4단계로 연결됩니다. **Claude Code → MCP 서버 → WebSocket 서버 (포트 3055) → Figma Desktop 플러그인.** 본인 PC 안에서 Figma Desktop 앱을 Claude 가 직접 조작합니다.
>
> 사용법은 간단합니다.
>
> **`figma 시작하자`** ← 라우터가 켜집니다.
>
> Figma 설치를 자동 감지해서 **작업 가능 업무 5개 메뉴**를 띄워줍니다. 본인은 '**1**' 누르면 시연 (카드뉴스 8장 와이어프레임) 자동 진행.
>
> 시연의 4단계 내부 동작 · ① 본인 Figma 에 열린 레퍼런스 카드뉴스를 `scan_text_nodes` + `get_styles` + `get_styled_text_segments` 로 분석 ② 분석 결과를 컬러·폰트·간격 토큰 시스템으로 정리 ③ Claude 가 텍스트로 8장 콘텐츠 기획 ④ 디자인 시스템 적용한 표지 1장 (`create_frame` + `create_text` × N) + `clone_node` × 7 (각 가로 좌표 다르게) + `set_multiple_text_contents` 한 번에 7장 텍스트 동시 교체.
>
> 그 다음이 본 클립의 진짜 가치입니다. **Figma 파일 우상단 Share 버튼 → 디자이너 이메일 추가 → 권한 부여.** 끝. 디자이너는 그 Figma 파일을 열어서 와이어프레임 위에 바로 디자인을 입힙니다. PPT 를 Figma 로 재작성하는 1시간이 사라집니다.
>
> 마케터는 Figma 코멘트로 실시간 피드백 · 디자이너 수정 · 같은 파일에서 즉시 확인. 메일 왕복 0회."

〔→ 05 마케터 가치〕

### 🎬 [4:00~4:35] 마케터 가치 3가지

〔슬라이드: 05 가치 3 카드〕

**🎙️ 강사 발화:**

> "Figma MCP 가 마케터한테 주는 가치 3가지입니다.
>
> **첫째, 기획안 작성 시간 단축** · PPT 의뢰서 1~2시간이 자연어 1분으로. 디자이너 재작성 1시간도 사라집니다.
>
> **둘째, 디자이너와 빠른 협업** · 같은 Figma 파일에서 비주얼 폴리시가 즉시 시작. 의뢰서 재작성 없음. 코멘트로 실시간 피드백. 의사결정 사이클이 반나절에서 즉시로.
>
> **셋째, 무제한 호출** · Figma 공식 호스팅의 한 달 6번 제약 없음. 의뢰 자료가 부족하면 무제한 변형·복제·생성.
>
> 한 줄로 정리하면 · **마케터와 디자이너가 같은 Figma 파일에서 협업을 시작한다.**"
