# 클립 2-1 실습 · Figma MCP · 원하는 디자인 카드뉴스 8장 자동 생성 (4단계 흐름)

## 목표 산출물

1. `claude-talk-to-figma-mcp` 설치 + WebSocket 서버 기동 + Figma Plugin 임포트
2. **원하는 디자인 카드뉴스 8장 시리즈 자동 생성** (4단계 흐름)
   ① 레퍼런스 카드뉴스 디자인 분석 → ② 디자인 시스템 구축 → ③ 카드뉴스 기획 → ④ 시스템 적용한 8장 제작
3. **디자이너 Share 권한 부여** → Figma 공동작업 시작

## 왜 이 4단계 흐름인가

마케터가 자연어로 "예쁜 카드뉴스 만들어줘" 라고 하면 결과가 그때그때 다릅니다. 본인이 원하는 톤·컬러·폰트·여백을 그대로 살리려면 **레퍼런스를 먼저 분석해 디자인 시스템으로 만들고, 그 시스템에 맞게 적용** 해야 합니다. 4단계 흐름은 talk-to-figma 의 분석 도구 (`scan_text_nodes`·`get_styles`·`get_styled_text_segments`) + 생성·수정 도구를 한 흐름에 연결합니다.

## 시연 · 원하는 디자인 카드뉴스 8장 (4단계 흐름)

### 사전 준비

- Figma Desktop 에 **레퍼런스 카드뉴스 파일** 1개 열어두기 (본인이 좋아하는 톤의 카드뉴스 1세트)
- 본 강의 시연용으로 무료 Figma Community 카드뉴스 템플릿 1개 복사해서 사용해도 OK

---

### 1단계 · 레퍼런스 카드뉴스 디자인 분석 (약 30초)

```
[역할] 너는 시니어 브랜드 디자이너야.

[요청] 현재 Figma 파일에 열린 레퍼런스 카드뉴스를 분석해줘:
  · 모든 텍스트 노드의 폰트·크기·색상·간격 추출
  · 사용된 컬러 팔레트 (배경·헤드라인·서브·CTA 별)
  · 폰트 종류·크기 단계 (헤드라인·서브카피·라벨)
  · 카드 레이아웃 (여백·패딩·요소 정렬)

[산출물] 분석 결과 마크다운 표
```

→ 호출 도구 : `get_document_info` + `scan_text_nodes` + `get_styles` + `get_styled_text_segments` + `get_node_info`. **약 30초.**

### 2단계 · 분석 결과를 디자인 시스템으로 구축 (약 30초~1분)

```
[요청] 위 분석 결과를 재사용 가능한 디자인 시스템으로 정리해줘:

[산출물 1] 마크다운 디자인 토큰 표
  · 컬러 토큰 (primary·secondary·background·text·CTA 5단계)
  · 폰트 토큰 (heading-XL·heading-L·body·caption 4단계)
  · 간격 토큰 (xs 4px / sm 8px / md 16px / lg 24px / xl 48px)
  · 카드 레이아웃 규칙 (프레임 크기·padding·요소 간격)

[산출물 2] (옵션) Figma 변수로 등록
  · 본인 파일에 컬러·폰트 변수 자동 생성
```

→ 호출 도구 : 도구 0 (Claude 가 텍스트로 시스템 정리). 옵션으로 `create_text` + `set_text_style_id` 활용. **약 30초~1분.**

### 3단계 · 카드뉴스 8장 콘텐츠 기획 (약 10~20초)

```
[역할] 너는 시니어 콘텐츠 기획자야.

[요청] 인스타용 카드뉴스 8장 시리즈를 기획해줘:
  · 주제 "1인 마케터를 위한 AI 자동화 7가지 팁"
  · 타겟 자영업자·1인 마케터
  · 톤 친근하지만 정중

[산출물] 8장 구조 (각 카드별 헤드라인·서브카피·CTA 1줄)
  - 1장 (표지) : 시리즈 후크
  - 2장 (팁 1) : MCP 설치 한 줄 명령
  - 3장 (팁 2) : 스킬로 반복 작업 자동화
  - 4장 (팁 3) : 에이전트 24시간 가동
  - 5장 (팁 4) : 데이터 자동 수집 (GA4·Sheets)
  - 6장 (팁 5) : 콘텐츠 자동 생성 (Figma·Buffer)
  - 7장 (팁 6) : 광고 분석 자동화 (Meta·Google Ads)
  - 8장 (결말) : 7가지 팁 통합 + 다음 단계
```

→ Claude 가 텍스트로 8장 기획안 생성. 도구 0개. **약 10~20초.**

### 4단계 · 디자인 시스템 적용한 8장 카드뉴스 제작 (약 1~2분)

```
[요청] 2단계 디자인 시스템을 그대로 적용해서 8장 카드뉴스를 만들어줘:

[표지 카드 (1장)]
  · 프레임 1080×1080 (배경 = 2단계 토큰의 primary)
  · 상단 라벨 "7가지 팁" (caption 폰트 토큰)
  · 중앙 헤드라인 (heading-XL 토큰, 3단계 기획의 1장 헤드라인)
  · 서브 카피 (body 토큰, 3단계 기획의 1장 서브카피)
  · 페이지 번호 "1 / 8" (caption 토큰)
  · 간격은 2단계 토큰 (xl 48px / lg 24px)

[본문 7장 (clone × 7)]
  · 표지 카드를 7번 복제 (x 좌표 1080, 2160, 3240, 4320, 5400, 6480, 7560)
  · 각 카드의 헤드라인·서브카피·페이지 번호를 3단계 기획의 2~8장으로 교체
  · 페이지 번호 "2 / 8" ~ "8 / 8"

[제약] 컬러·폰트·간격은 모두 2단계 디자인 시스템 토큰 사용
```

→ 호출 도구 : `create_frame` + `create_text` × 4 + `set_fill_color` × N + `set_font_name` + `set_font_size` + `set_auto_layout` + `clone_node` × 7 + `set_multiple_text_contents` 1회. **약 1~2분.**

**총 약 3분 · 결과 : 본인이 원하는 디자인 톤으로 통일된 8장 카드뉴스 시리즈.**

---

### 5단계 · Figma Share 권한 → 디자이너 공동작업 시작

```
1. Figma 파일 우상단 Share 버튼
2. 디자이너 이메일 입력 + "can edit" 권한
3. 링크 복사 → 디자이너에게 전달
```

→ 디자이너는 같은 파일에서 와이어프레임 위에 비주얼 폴리시 시작 (이미 본인 디자인 시스템이 적용되어 있어 톤·맥락 일치). PPT 를 Figma 로 재작성하는 시간이 사라집니다.

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| `Plugin not found` | Figma Desktop → Plugins → Development → Claude MCP Plugin 실행 |
| `Channel ID mismatch` | 플러그인 재시작 후 새 ID 복사하고 다시 연결 |
| 1단계 분석 시 텍스트 일부만 추출 | `scan_text_nodes` 의 chunkSize 옵션 명시 (기본 10) |
| `Font not found` (한글) | `load_font_async` 자동 호출 시도. 그래도 안 되면 Pretendard 본인 PC 설치 |
| Port 3055 충돌 | `lsof -i :3055` → 점유 프로세스 종료 |
| `Permission denied` | Figma 새 파일 생성 (본인 소유) 또는 편집자 권한 받기 |
| 카드 8장이 같은 자리에 겹침 | clone 시 x 좌표 명시 ("1080, 2160, 3240, 4320, 5400, 6480, 7560") |
| 4단계 적용 시 시스템 토큰 무시됨 | 2단계 토큰 표를 Claude 메시지에 다시 첨부 (메모리 한계 보완) |
| Auto Layout 미작동 | `set_auto_layout` 명시 요청 (직접 호출) |
| 디자이너가 공동작업 불가 | Figma 파일 Share → 디자이너 이메일 추가 + "can edit" 권한 |
