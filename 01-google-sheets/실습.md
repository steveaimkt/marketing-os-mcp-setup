# 클립 1-2 실습 · Google Sheets MCP 설치 + 첫 분석 리포트

## 실습 한 줄 요약

`/mcp설치-google-sheets` 스킬을 호출해 OAuth 5단계를 15~20분 안에 설치하고, `/start` 한 줄로 5단계 자동 리포트를 생성한다.

## 사전 준비

본 클립 시작 전 다음 4가지를 확인하세요:

```
□ Node.js 18 이상       → 터미널에서 `node --version`
□ Google 계정 1개        → console.cloud.google.com 로그인 가능
□ 마케팅용 시트 1개      → 분석할 데이터 시트 (없으면 샘플로 시작)
□ Chrome 또는 Safari    → OAuth 브라우저 인증용
```

API 키는 발급하지 않음 (OAuth 방식). 본 실습은 모두 무료.

## 단계별 실행

### Step 0. 스킬 호출

Claude Code 에 한 줄 입력:

```
Google Sheets MCP 설치하자
```

자동 호출되는 스킬: `/mcp설치-google-sheets`

이후 모든 단계가 스킬 안내에 따라 자동 진행됩니다. 사용자가 직접 작업하는 곳은 **Google Cloud 설정 + 브라우저 인증** 두 곳.

### Step 1. Google Cloud OAuth 자격증명 발급 (10분)

사용자 직접 작업:

```
① console.cloud.google.com 접속 → 새 프로젝트 'sheets-mcp' 생성
② API 및 서비스 > 라이브러리 > 'Google Sheets API' 사용 설정
③ OAuth 동의 화면 > User Type '외부' > 앱 이름 'Sheets MCP'
④ 테스트 사용자 탭 > 본인 이메일 추가 ⚠️ 필수
⑤ 사용자 인증 정보 > OAuth 클라이언트 ID > 데스크톱 앱 > 만들기
⑥ JSON 다운로드 → 'oauth_credentials.json' 으로 이름 변경
```

다운로드한 JSON 파일에 `"installed"` 키가 있으면 정상. `"web"` 이면 "데스크톱 앱" 이 아닌 "웹 애플리케이션" 을 선택한 것 · 다시 생성 필요.

### Step 2. MCP 서버 설치 (3분)

Claude 자동 처리:

```bash
# 1) marketing-os/ 하위에 mcp-server 폴더 생성
mkdir -p mcp-server && cd mcp-server

# 2) oauth_credentials.json 을 mcp-server/ 에 배치
mv ~/Downloads/oauth_credentials.json ./

# 3) MCP 서버 코드 + 의존성 설치
npm init -y
npm install @modelcontextprotocol/sdk googleapis
```

설치되는 패키지:
- `@modelcontextprotocol/sdk` · MCP 프로토콜 표준
- `googleapis` · Google Sheets API v4 클라이언트

### Step 3. Claude Code 에 MCP 등록 (2분)

Claude 자동 처리: `marketing-os/.mcp.json` 에 다음 항목 추가

```json
{
  "mcpServers": {
    "google-sheets": {
      "command": "node",
      "args": ["${CLAUDE_PROJECT_DIR}/mcp-server/index.js"]
    }
  }
}
```

또는 사용자 전역 등록 (`~/.claude/settings.json`):

```bash
claude mcp add google-sheets \
  --transport stdio \
  -- node "/절대경로/mcp-server/index.js"
```

절대경로 확인: `cd mcp-server && pwd`

### Step 4. 브라우저 인증 (1회만, 3분)

Claude Code 완전 종료 → 재시작 시 브라우저 자동 열림:

```
① Google 계정 선택 (Step 1 ④ 에서 등록한 이메일)
② "이 앱은 확인하지 않았습니다" → 고급 → 이동
③ 스프레드시트 읽기/쓰기 권한 → 허용
④ "✅ 인증 완료!" 표시 → 브라우저 닫기
```

`mcp-server/token.json` 자동 생성되면 성공. 토큰 만료 시 자동 갱신.

### Step 5. 연결 확인 + 첫 명령

Claude Code 에서:

```
스프레드시트 시트 목록 보여줘
```

시트 이름·행 수·열 수가 표시되면 설치 완료.

## 첫 결과물 · `/start` 5단계 자동 리포트

```

Step 1/5 · 데이터 탐색     → batch_read_sheet 로 3 시트 동시 (10초)
Step 2/5 · 데이터 분석     → 채널별 매출 + ROAS + 재고 분석  (30~60초)
Step 3/5 · 시트 기록       → '분석결과' 시트 자동 생성 + 서식 (30초)
Step 4/5 · 추가 분석       → 사용자 질문 추가 (선택)
Step 5/5 · 최종 리포트     → 새 스프레드시트 생성 + URL 공유  (1~2분)

[결과물]
🔗 '주간 종합 리포트' 스프레드시트 (4개 시트)
   - 총매출 ₩47,004,000
   - ROAS 857%
   - 긴급재고 3건
   - URL 공유로 팀 즉시 열람 가능
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `액세스 차단됨` | 테스트 사용자 미등록 | OAuth 동의 화면 > 테스트 사용자 탭에 본인 이메일 추가 |
| `Cannot find module @modelcontextprotocol/sdk` | npm install 누락 | `cd mcp-server && npm install` 재실행 |
| MCP 도구 안 보임 (`mcp__google-sheets__*` 미노출) | settings.json 경로 오타 또는 재시작 안 함 | `claude mcp list` 로 등록 확인 + Claude Code 완전 종료 후 재시작 |
| `Quota exceeded for quota metric 'Read requests'` | 분당 60회 초과 | 60초 대기 후 재시도 · 또는 `batch_read_sheet` 사용 |
| 토큰 만료 (`Token has been expired`) | 60일 이상 미사용 | `rm mcp-server/token.json` 후 재시작 → 재인증 |
| `Permission denied` (시트 접근 불가) | 본인 소유가 아닌 시트 | 시트 공유 권한에 본인 이메일 추가 또는 본인 시트로 시작 |
| OAuth 후 브라우저 안 닫힘 | 자동 종료 실패 | 수동으로 닫고 Claude 재시작 |
