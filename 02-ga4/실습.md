# 클립 1-3 실습 · GA4 MCP 설치 및 헬스 체크

## 목표 산출물

- `.env` 에 `GA_PROPERTY_ID` 등록 완료
- gcloud ADC 발급 (사용자 본인 계정 OAuth · 조직 정책 우회)
- "지난 7일 채널별 세션 수" 명령에 표 응답
- Part 7 에이전트가 사용할 기초 데이터 추출 검증

## 사전 준비물

```
□ Google Analytics 4 속성 1개 (UA 아닌 GA4)
□ 본인이 그 GA4 속성에 접근 권한 있는 Google 계정 (보통 본인 운영 속성이면 자동)
□ macOS / Linux / Windows + Node.js 18+
□ GA4 속성에 지난 28일 트래픽 데이터 존재
```

> ⚡ **Service Account JSON 키 발급 불필요**. 본인 Google 계정 ADC (Application Default Credentials) 만으로 작동. Google Workspace 조직의 `iam.disableServiceAccountKeyCreation` 정책도 우회.

## 단계별 명령 (스킬 자동 진행 + 사용자 확인)

### Step 1 · gcloud CLI 설치 + ADC 발급 (사용자 5분)

#### 1.1 gcloud CLI 설치 (macOS / Linux · arm64 / x86)

```bash
# 자동 다운로드 + 설치 (Claude 가 안내)
curl -sSL "https://dl.google.com/dl/cloudsdk/channels/rapid/downloads/google-cloud-cli-darwin-arm.tar.gz" -o ~/gcloud-cli.tar.gz
cd ~ && tar -xzf gcloud-cli.tar.gz
~/google-cloud-sdk/install.sh --quiet --usage-reporting=false --command-completion=true --path-update=true --rc-path=$HOME/.zshrc
exec -l $SHELL  # 또는 새 터미널

# 검증
gcloud --version
# → "Google Cloud SDK 569.0.0" 정도 출력
```

Windows (PowerShell):
```powershell
(New-Object Net.WebClient).DownloadFile("https://dl.google.com/dl/cloudsdk/channels/rapid/downloads/google-cloud-cli-windows-x86_64.zip", "$env:Temp\gcloud.zip")
Expand-Archive -Path "$env:Temp\gcloud.zip" -DestinationPath $env:UserProfile
& "$env:UserProfile\google-cloud-sdk\install.bat" --quiet
```

#### 1.2 ADC 발급 (사용자 OAuth 1회)

```bash
# 본인 OAuth client (marketing-os/mcp-server/oauth_credentials.json) 활용
# - Google 기본 client ID 는 Workspace 조직에 차단되므로 본인 것 사용 필수
gcloud auth application-default login \
  --client-id-file="${CLAUDE_PROJECT_DIR}/mcp-server/oauth_credentials.json" \
  --scopes=https://www.googleapis.com/auth/analytics.readonly,https://www.googleapis.com/auth/cloud-platform

# → 브라우저 자동 열림 → 본인 Google 계정 로그인 → "허용"
# → ~/.config/gcloud/application_default_credentials.json 자동 생성
# → "Credentials saved to file" 메시지 확인
```

> ⚠️ Google 기본 client ID 로 직접 발급 시 **"차단된 앱"** 오류 (Google Workspace 보안 정책). 반드시 본인 OAuth credentials 파일 (Google Sheets MCP 설치 시 발급된 것) 활용.

### Step 2 · GA4 속성 ID 확인 + `.env` 등록 (사용자 2분)

```
1. analytics.google.com 접속 (위 ADC 와 같은 계정)
2. 본인 GA4 속성 선택
3. 좌하단 ⚙ "관리"
4. 우측 "속성" 컬럼 > "속성 설정"
5. 우상단 영역에 "속성 ID" 라벨 + 9자리 숫자 (예: 471063826)
6. 9자리 숫자 복사 (G-XXX 형식 측정 ID 아닙니다)
```

`.env` 에 추가:

```bash
GA_PROPERTY_ID=471063826   # 본인 GA4 속성 ID (9자리)
```

> ⚠️ 변수명은 **`GA_PROPERTY_ID`** (GA4_ 아닌 GA_). `mcp-server-ga4` 패키지가 요구하는 정확한 이름.

### Step 3 · `.mcp.json` 등록 + 헬스 체크 (Claude 자동 1분)

이미 marketing-os/.mcp.json 에 등록된 항목 (확인만):

```json
"ga4": {
  "command": "npx",
  "args": ["-y", "mcp-server-ga4"],
  "env": {
    "GOOGLE_APPLICATION_CREDENTIALS": "${HOME}/.config/gcloud/application_default_credentials.json",
    "GA_PROPERTY_ID": "${GA_PROPERTY_ID}"
  }
}
```

헬스 체크:
```bash
claude mcp list | grep ga4
# → ga4: npx -y mcp-server-ga4 - ✓ Connected
```

`✓ Connected` 가 아니면:
- ADC 파일 존재 확인: `ls ~/.config/gcloud/application_default_credentials.json`
- `GA_PROPERTY_ID` env 확인: `grep GA_PROPERTY_ID .env`

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `차단된 앱` (브라우저) | Google 기본 client ID 가 Workspace 조직에 차단 | `--client-id-file=` 옵션으로 본인 OAuth credentials 사용 |
| `PERMISSION_DENIED` | 본인 계정이 GA4 속성에 접근 권한 없음 | analytics.google.com > 관리 > 속성 액세스에 본인 이메일 권한 확인 |
| `GA_PROPERTY_ID is not set` | 환경변수 이름 오타 (`GA4_PROPERTY_ID` 잘못 사용) | `.env` 의 변수명을 `GA_PROPERTY_ID` 로 수정 (GA4_ 아님) |
| `INVALID_ARGUMENT` property ID | `properties/` 접두사 중복 | 9자리 숫자만 (MCP 자동 prefix) |
| 응답이 비어 있음 | 데이터 범위에 트래픽 없음 | 기간을 `last_90_days` 로 늘려 재시도 |
| ADC 파일 없음 | gcloud auth 미실행 | Step 1.2 명령 재실행 |
