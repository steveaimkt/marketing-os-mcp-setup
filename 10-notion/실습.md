# 클립 4-1 실습 · Notion MCP 활성화 + 콘텐츠 캘린더 30개 자동 등록

## 실습 한 줄 요약

`/mcp설치-notion` 스킬을 호출해 Claude.ai 통합을 5분 안에 활성화하고, 5월 콘텐츠 캘린더 30개 카드를 한 줄 명령으로 자동 등록한다.

## 사전 준비

본 클립 시작 전 다음 4가지를 확인하세요:

```
□ Notion 무료 계정             → notion.so 가입
□ Claude.ai 계정 + 구독        → claude.ai (Notion 통합은 claude.ai 구독 인증 필요)
□ 본인 워크스페이스 1개        → 새 페이지 만들 권한
□ Chrome 또는 Safari           → OAuth 인증용 브라우저
```

본 실습은 모두 무료. API key 발급 없음 (OAuth 통합 방식).

## 단계별 실행

### Step 0. 스킬 호출

Claude Code 에 한 줄 입력:

```
Notion MCP 설치하자
```

자동 호출되는 스킬: `/mcp설치-notion`

이후 모든 단계가 스킬 안내에 따라 자동 진행됩니다. 사용자가 직접 작업하는 곳은 **OAuth 인증 1번** 과 **페이지 공유 (Add connections) 1~2번**.

### Step 1. Claude.ai 인증 모드 확인 (1분)

사용자 직접 작업:

```
① 터미널에서 `/status` 명령 또는 Claude Code 메뉴 > 인증 확인
② "Claude.ai 구독" 모드 확인 (Anthropic API key 모드가 아님)
③ 만약 API key 모드면: 환경변수 ANTHROPIC_API_KEY 제거 후 `/login` → claude.ai 선택
```

Notion MCP 는 claude.ai 통합으로만 작동. API key 모드에서는 도구가 안 보입니다.

### Step 2. Notion 통합 활성화 (사용자 직접 · 2분)

브라우저에서 진행:

```
① claude.ai 로그인 → 우상단 프로필 → Settings
② 좌측 메뉴 "Integrations" 또는 "Connectors" 선택
③ Notion 항목 찾기 → "Connect" 또는 "활성화" 클릭
④ Notion OAuth 창 자동 열림 → 본인 워크스페이스 선택
⑤ Claude 가 접근할 권한 범위 확인 (검색·조회·작성·수정)
⑥ "Allow access" 클릭 → 통합 완료
```

활성화 후 `mcp__claude_ai_Notion__*` 도구 16개가 자동 노출됩니다 (재시작 불필요할 수도, 일부 환경에서는 필요).

### Step 3. 노션 페이지 공유 (사용자 직접 · 2분 · 페이지마다 1회)

```
① Notion 에서 작업할 페이지 (예: 콘텐츠 캘린더 DB) 열기
② 우상단 ··· 메뉴 → "Add connections" 또는 "연결 추가"
③ "Claude" 검색 → 선택 → "Confirm" 클릭
④ 해당 페이지와 모든 하위 페이지 접근 권한 부여됨
```

⚠️ **중요**: Notion 의 보안 모델상 통합이 명시적으로 공유된 페이지만 접근 가능. 워크스페이스 전체 접근이 아님. 새 페이지·DB 마다 1회 공유 필요.

권장: 워크스페이스 최상위 "Marketing" 같은 모폴더 페이지 1개에 통합 공유 → 하위 모든 페이지 자동 접근.

### Step 4. Claude Code 헬스 체크 (1분)

Claude Code 에서 (필요시 재시작 후) 다음 명령:

```
노션 워크스페이스에서 콘텐츠 캘린더 DB 찾아줘
```

성공 시 응답:

```
✅ Notion 통합 확인. 검색 결과:
  1. "콘텐츠 캘린더 2026" · DB · 채널 캘린더 뷰
     URL: notion.so/abc123...
  2. "콘텐츠 백로그" · 페이지
     ...

도구 16종 사용 가능: notion-search, notion-fetch, notion-create-pages, ...
```

내부적으로 `mcp__claude_ai_Notion__notion-search` 호출됨.

## 첫 결과물 · 5월 콘텐츠 캘린더 30개 자동 등록

### 명령

```
콘텐츠 캘린더 DB 에 5월 한 달 카드 30개 등록해줘.

카드 스펙:
- 제목: 자동 생성 (5월 신학기 캠페인 컨셉)
- 날짜: 5월 1일 ~ 5월 31일 (주 5회 페이스 · 평일 위주)
- 채널: Instagram (15) · Facebook (8) · X (5) · LinkedIn (2)
- 상태: "draft" (모두 초안)
- 담당자: 자기 (admin 또는 본인 이메일)
- 카테고리: 캠페인 (15) · 일반 콘텐츠 (10) · 후기·UGC (5)

브랜드 톤: 친근·전문성 균형 · 신학기·새 시작 키워드
```

### 자동 실행

Claude 가 도구를 순차 호출:

```
1. mcp__claude_ai_Notion__notion-search
   → "콘텐츠 캘린더" DB 검색 → URL 확인

2. mcp__claude_ai_Notion__notion-fetch
   → DB 스키마 조회 (제목·날짜·채널·상태·담당자·카테고리 속성 확인)

3. Claude 가 30개 카드 데이터 생성
   - 5월 평일 캘린더 매핑 (총 22 영업일 + 주말 일부)
   - 채널·카테고리 분배
   - 제목 30개 자동 작성 (신학기·새 시작 키워드)

4. mcp__claude_ai_Notion__notion-create-pages × 30번 호출
   - 각 호출: parent (DB URL) + properties (속성 6개)
   - 페이지 ID 반환

5. 결과 표 출력:
   ✅ 30개 카드 등록 완료
   | # | 날짜 | 채널 | 제목 | URL |
   | 1 | 5/2 (목) | IG | 신학기, 책상 위 한 줄 응원 | notion.so/... |
   ...
```

소요 시간: **약 5분** (수동 입력 1~2시간).

## 두 번째 결과물 · 광고 리포트 아카이브 DB 자동 생성

```
광고 리포트 아카이브 DB 새로 만들어줘.

스키마:
- 제목 (Title)
- 날짜 (Date · 발행일)
- 매체 (Select · Meta·Google·네이버·Kakao)
- ROAS (Number · 소수점 2자리)
- 인사이트 (Text · 3~5문장)
- 상태 (Select · 검토중·승인·보관)
- 첨부 (URL · HTML 리포트 링크)
- 담당자 (Person)

위치: "Marketing > 광고" 하위 페이지.
뷰: 기본 캘린더 뷰 + "이번 달 ROAS 높은 순" 보조 뷰.
```

소요 시간 약 1~2분. Part 6 광고 클립 에이전트들이 이 DB 에 매주 자동 게시.

## 세 번째 결과물 · 브랜드 가이드 페이지 자동 작성

```
"브랜드 가이드라인" 페이지 만들어줘.

섹션:
1. 브랜드 보이스 (3 키워드 + 1줄 설명)
2. 톤 (Casual · Professional 비중)
3. 페르소나 (이상적 고객 1명 묘사)
4. 금기어 (사용 금지 5개)
5. 자주 쓰는 표현 (Do 10개 · Don't 5개)
6. 예시 카피 3종 (인스타·페북·LinkedIn 톤별)

위치: "Marketing > 콘텐츠" 하위.
```

소요 시간 약 2분. Part 5 의 `brand-guideline` 에이전트가 이 패턴으로 자동 생성.

## 마케터 5패턴 · 정기 운영 결합

```
[역할]
D2C 브랜드 콘텐츠 매니저 + 광고 운영 어시스턴트

[입력]
- 콘텐츠 캘린더 DB URL (월간 30개 카드 등록 대상)
- 광고 리포트 아카이브 DB URL (주간 게시 대상)
- 브랜드 가이드 페이지 (참조용)

[산출물]
매주 월요일 자동 수행:
  ① 광고 리포트 카드 1개 (notion-create-pages) · 매체별 ROAS·인사이트
  ② 콘텐츠 캘린더 다음 주 카드 5~7개 등록 (브랜드 톤 적용)
  ③ Discord #marketing 채널에 Notion URL 발송

[제약]
- DB 속성명 정확히 일치 (notion-fetch 로 사전 확인)
- 페이지 본문 마크다운 형식
- 브랜드 가이드 페이지 톤·금기어 준수
- 한국어 페이지 제목 (검색 안 되면 영문 키워드 병행 추가)

[검증]
- 30개 카드 등록 시 누락 0건
- DB 뷰의 캘린더에 30개 모두 표시
- 브랜드 톤 일관성 (가이드 페이지의 금기어 0건)
- 생성 시간 10분 이내
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `mcp__claude_ai_Notion__*` 도구 안 보임 | Claude.ai 통합 미활성화 또는 API key 모드 | claude.ai > Settings > Integrations > Notion > Connect + `/login` 으로 claude.ai 인증 |
| `Page not found` | 통합에 페이지 미공유 | 해당 페이지 우상단 ··· > Add connections > Claude 선택 |
| 워크스페이스 검색 결과 0건 | 통합에 어떤 페이지도 미공유 | 최상위 페이지 (예: "Marketing") 1개에 공유하면 하위 자동 |
| `Property does not exist` | DB 속성명 오타 또는 영/한 불일치 | `notion-fetch <DB URL>` 로 정확한 속성명·타입 먼저 확인 |
| 한글 페이지 제목 검색 안 됨 | 검색 인덱스 지연 또는 토큰화 정확도 | URL 직접 사용 또는 영문 키워드 병행 (예: "캠페인 campaign") |
| Calendar 뷰에 카드 안 나옴 | Date 속성 누락 또는 뷰 필터 차단 | 카드의 Date 속성 채워졌는지 확인 + 뷰 필터 조건 점검 |
| 30개 카드 등록 중 일부 실패 | 일시 rate limit (Notion API 분당 약 3 req/sec) | 자동 재시도 (스킬 내장). 수동 시 1초 간격으로 재호출 |
| 페이지 삭제 안 됨 | MCP 가 휴지통 이동만 지원 | Notion UI 에서 영구 삭제 (휴지통 비우기) |

## 응용 과제

1. 본인 워크스페이스의 페이지 1개 (예: "테스트 페이지") 만들고 통합에 공유 → "내 워크스페이스 페이지 찾아줘" 로 검색 확인
2. 콘텐츠 캘린더 DB 본인 1개 만들고 (또는 기존 사용) → "5월 첫 주 콘텐츠 5개 카드 등록" 1회 시연
3. `notion-fetch` 로 DB 스키마 가져온 후 → AI 가 자동으로 카드 속성 채우기
4. **Part 5 콘텐츠·카피 클립에서 `brand-guideline` 에이전트가 본 MCP 를 자동 호출** · 미리 "브랜드 가이드라인" 페이지 위치 정해두기
