# 클립 3-2 실습 · Meta·Google Ads MCP 설치 (공식 2종)

## 목표 산출물

- Meta hosted MCP 등록 + OAuth 인증 (Claude.ai 자동 갱신)
- Google 공식 MCP 등록 + Developer Token 1개 발급 (1~2일 승인 대기)
- 매체 통합 KPI 표 1건 (Meta + Google)

## 사전 준비물

```
□ Node.js 18+ · uv (uvx 명령)
□ Meta Business Manager + 본인 광고 계정 1개
□ Google Ads Manager 계정 1개 (없으면 무료 생성, 광고비 0원)
□ gcloud CLI + ADC 발급 (GA4 와 공유, 이미 발급된 상태)
□ 광고 캠페인 1개 이상 운영 중 (시연용, 선택)
```

## 트리거 1줄

```
Meta Google Ads MCP 설치하자
```

→ `/mcp설치-ads` 스킬 자동 호출.

## 단계별 명령

### Part A · Meta Ads (1단계 · 약 1~2분) ⭐ 공식 hosted MCP

#### A1. Meta hosted MCP 등록 (Claude 자동 30초)

```bash
claude mcp add --transport http --scope user meta-ads https://mcp.facebook.com/ads

# 검증
claude mcp list | grep meta-ads
# → meta-ads: https://mcp.facebook.com/ads (HTTP) - ! Needs authentication
```

`! Needs authentication` 상태가 정상. 다음 단계에서 인증.

#### A2. OAuth 인증 (사용자 1회 · 1분)

```
1. Claude Code 완전 종료 (⌘Q on macOS)
2. marketing-os 디렉토리에서 재시작
3. 다음 명령 입력:

   "내 Meta 광고 캠페인 목록 보여줘"

4. 브라우저 자동 열림 → Facebook 로그인
5. 5개 OAuth scope 허용:
   - ads_management
   - ads_read
   - catalog_management
   - business_management
   - pages_show_list
6. 자동 닫힘 → 인증 완료
```

OAuth 토큰은 Claude.ai 가 자동 관리·갱신. System User Token / `.env` 변수 발급 작업 **모두 불필요**.

#### A3. 헬스 체크 (자동 30초)

```bash
claude mcp list | grep meta-ads
# → meta-ads: ... - ✓ Connected
```

또는 자연어:
```
사용자: "Meta 광고 도구 목록 보여줘"
→ Claude: mcp__meta-ads__* 도구 자동 노출
```

---

### Part B · Google Ads (2단계 · 약 5분 + 승인 1~2일) ⭐ 공식 MCP

#### B1. Developer Token 신청 (사용자 직접 · 5분 + 1~2일 대기)

```
1. ads.google.com (wmbb.kr 계정 로그인)

2. Manager 계정 (MCC) 필요:
   - 일반 광고 계정으로는 API Center 접근 불가
   - 없으면 https://ads.google.com/intl/ko_kr/home/tools/manager-accounts/
     에서 무료 생성 (광고비 0원, 약 2분)

3. Manager 계정 안에서:
   상단 ⚙ "도구" > "설정" > "API 센터"
   또는 https://ads.google.com/aw/apicenter

4. "Developer Token 신청" / "Apply for Basic access" 클릭

5. 신청 양식 작성:
   - 회사명: WMBB
   - 회사 웹사이트: wmbb.kr
   - 사용 목적 (영문 권장):
     "Internal marketing automation and analytics for our own
      Google Ads accounts via Google's official MCP server
      (github.com/googleads/google-ads-mcp). Read-only access only."
   - 예상 호출량: ~1,000 calls/day
   - 데이터 저장: "Stored locally, not shared"

6. 제출 → "Pending review" 상태
7. 1~2 영업일 이메일 알림 → 22자 Developer Token 발급
```

#### B2. MCP 등록 (Claude 자동 · 1분)

승인 받은 Developer Token 을 받으면:

```bash
# .env 1줄만 추가 (Claude 자동)
echo "GOOGLE_ADS_DEVELOPER_TOKEN=22자_토큰" >> .env
```

`.mcp.json` 의 `google-ads` 항목 (이미 등록됨, 확인만):

```json
"google-ads": {
  "command": "uvx",
  "args": [
    "--from",
    "git+https://github.com/googleads/google-ads-mcp.git",
    "google-ads-mcp"
  ],
  "env": {
    "GOOGLE_PROJECT_ID": "marketing-os-497122",
    "GOOGLE_ADS_DEVELOPER_TOKEN": "${GOOGLE_ADS_DEVELOPER_TOKEN}"
  }
}
```

> ⚡ **ADC 는 GA4 와 공유**. 별도 OAuth Client ID/Secret/Refresh Token/Customer ID 발급 작업 불필요. `~/.config/gcloud/application_default_credentials.json` (GA4 클립에서 발급) 자동 활용.

#### B3. 헬스 체크 (Claude Code 재시작 후 · 30초)

```bash
claude mcp list | grep google-ads
# → google-ads: uvx --from git+https://...google-ads-mcp - ✓ Connected
```

또는 자연어:
```
사용자: "내 Google 광고 계정 보여줘"
→ Claude: list_accessible_customers 자동 호출 → 광고 계정 목록 응답
```

### Part C · 첫 결과물 (1~2분)

```
사용자 명령:
"어제 Meta + Google 광고 통합 성과 표로 + 예산 재배분 안"

자동 진행:
1. Claude → Meta MCP: 어제 광고비·노출·CTR·전환·CPA·ROAS 조회

3. Claude → 두 데이터 통합 + 캠페인별 정렬

결과: 6 캠페인 통합 표 + 예상 추가 매출
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| Meta `Needs authentication` 지속 | OAuth 미인증 | Claude Code 재시작 후 "Meta 광고 캠페인 보여줘" 입력 |
| Meta `Permission denied` | 5 scope 중 일부 거부 | OAuth 재인증 시 5 scope 모두 허용 |
| Google `Developer token not approved` | Pending | 1~2일 승인 대기 |
| `An executable named ... not provided` | uvx 명령 오타 | `--from git+https://github.com/googleads/google-ads-mcp.git google-ads-mcp` 형식 확인 |
| `mcp__google-ads__*` 도구 안 보임 | uv 미설치 | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| `ADC not found` | gcloud auth 미실행 | GA4 클립에서 발급된 ADC 활용 또는 재실행 |
| `차단된 앱` (Google) | 기본 client ID 차단 | 본인 OAuth credentials 활용 (Sheets 클립 발급분) |
