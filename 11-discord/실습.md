# 클립 4-2 실습 · Discord MCP 설치 + 자동화 알림 봇 + 승인 워크플로

## 실습 한 줄 요약

`/mcp설치-discord` 스킬을 호출해 Discord Bot Token 발급 + 서버 초대 + `.mcp.json` 등록을 10~15분 안에 완료하고, 첫 자동화 봇 (주간 리포트 + 승인 reaction 워크플로) 1건을 시연한다.

## 사전 준비

본 클립 시작 전 다음 5가지를 확인하세요:

```
□ Node.js 18 이상            → 터미널에서 `node --version`
□ Discord 무료 계정          → discord.com 가입
□ 본인 Discord 서버 1개     → 없으면 새로 생성 (좌측 + 버튼 · 2분)
□ Discord Developer Portal  → discord.com/developers (계정 동일)
□ Chrome 또는 Safari        → OAuth 인증 + Token 발급용
```

본 실습은 모두 무료 (Discord 무료 + 개발자 봇 무제한).

## 단계별 실행

### Step 0. 스킬 호출

Claude Code 에 한 줄 입력:

```
Discord MCP 설치하자
```

자동 호출되는 스킬: `/mcp설치-discord`

이후 모든 단계가 스킬 안내에 따라 자동 진행됩니다. 사용자가 직접 작업하는 곳은 **Developer Portal 에서 봇 생성 + Token 발급 + 서버 초대** 세 가지.

### Step 1. Discord 서버 준비 (사용자 직접 · 2분)

이미 서버 있으면 스킵.

```
① Discord 좌측 사이드바 + 버튼 → "Create My Own"
② 서버 이름: "marketing-os" 또는 자유
③ 채널 4개 추천 (자동 생성 추가 가능):
   - #general (기본)
   - #marketing (주간 리포트·캠페인 알림)
   - #cs-inbox (고객 응대 큐)
   - #approvals (승인 워크플로)
```

### Step 2. Discord 봇 생성 + Token 발급 (사용자 직접 · 5분)

브라우저에서 진행:

```
① discord.com/developers/applications 접속 → 로그인
② "New Application" 클릭 → 이름: "Marketing OS Bot"
③ 좌측 메뉴 "Bot" 선택
④ "Reset Token" 클릭 → Token 복사 ⚠️ 한 번만 표시
   토큰 형식: 영문+숫자 약 70자 (점 2개 포함)
⑤ Privileged Gateway Intents 모두 활성화 (3개):
   - PRESENCE INTENT
   - SERVER MEMBERS INTENT
   - MESSAGE CONTENT INTENT ★ 필수 (메시지 읽기용)
⑥ "Save Changes" 클릭
```

### Step 3. 봇을 본인 서버에 초대 (사용자 직접 · 3분)

```
① 좌측 메뉴 "OAuth2" 선택
② "URL Generator" 클릭
③ Scopes 체크: ✅ bot, ✅ applications.commands
④ Bot Permissions 체크 (필수 7개):
   ✅ Send Messages
   ✅ Read Message History
   ✅ Manage Messages
   ✅ Add Reactions
   ✅ Use Slash Commands
   ✅ Embed Links
   ✅ Attach Files
⑤ 화면 하단 생성된 URL 복사
⑥ 새 탭에서 URL 열기 → 본인 서버 선택 → Authorize
⑦ "✅ Bot added" 표시 → Discord 서버 좌측에 봇 온라인 표시
```

### Step 4. Guild ID 복사 (사용자 직접 · 1분)

```
① Discord 본인 서버 우클릭 → "Server Settings"
② 좌측 메뉴 "Widget" 선택
③ "Server ID" 항목의 ID 복사 (숫자 약 18~19자)

또는 개발자 모드 활용:
① Discord Settings (좌하단 톱니바퀴) > Advanced > Developer Mode ON
② 서버 이름 우클릭 → "Copy Server ID"
```

### Step 5. .env 등록 (Claude 자동 · 1분)

Claude 자동 처리:

```bash
echo "DISCORD_TOKEN=발급받은_토큰" >> .env
echo "DISCORD_GUILD_ID=서버_ID" >> .env
```

### Step 6. .mcp.json 등록 (Claude 자동 · 1분)

Claude 자동 처리: `marketing-os/.mcp.json` 의 `mcpServers` 에 추가

```json
"discord": {
  "_part": "2 Ch 4-2 자동화 알림·승인 봇",
  "command": "npx",
  "args": ["-y", "mcp-discord"],
  "env": {
    "DISCORD_TOKEN": "${DISCORD_TOKEN}",
    "DISCORD_GUILD_ID": "${DISCORD_GUILD_ID}"
  }
}
```

### Step 7. Claude Code 재시작 + 헬스 체크 (1분)

Claude Code 완전 종료 → 재시작. 다음 명령으로 검증:

```
디스코드 서버 채널 목록 보여줘
```

성공 시 응답:

```
✅ Discord 봇 연결 확인. 서버 "marketing-os" 채널 4개:
  - #general (text)
  - #marketing (text)
  - #cs-inbox (text)
  - #approvals (text)

봇 상태: 온라인 · 권한: Send/Read/React/Manage 7종
사용 가능 도구 8종: send_message, read_messages, list_messages, ...
```

내부적으로 `mcp__discord__list_channels` 호출됨.

## 첫 결과물 1 · 유튜브 채널 분석 → 디스코드 자동 보고

본 시연의 핵심은 **Discord MCP 가 매개** 라는 점. 슬래시 명령 또는 채널 멘션으로 트리거 → 다른 MCP 호출 → 결과를 디스코드로 자동 발송.

**사전 조건**: YouTube Data MCP 사전 설치 (Part 2 / 2-2 클립). 본인 유튜브 채널 1개.

### 명령

디스코드 `#marketing` 채널에서:

```
@봇 우리 채널 지난 7일 분석해줘

또는 슬래시 명령:
/youtube-analyze weekly
```

### 자동 실행

```
1. Discord MCP · 채널 멘션 이벤트 수신
   - 이벤트: messageCreate
   - 멘션 대상: 본 봇
   - 트리거 키워드: "분석"

2. mcp__youtube-data__get_channel_stats 호출
   - 채널 ID: 본인 채널
   - 기간: 지난 7일

3. mcp__youtube-data__list_recent_videos 호출
   - 상위 5개 영상 조회수·구독자 변화

4. mcp__youtube-data__get_comments 호출
   - 영상별 댓글 → Claude 가 감정 분석 (긍정·부정·중립)

5. mcp__discord__send_message 호출
   - 채널: #marketing
   - embed:
     * 제목: 📺 W19 채널 성과 요약
     * 필드: 구독자 변화 · 조회수 · 댓글 감정
     * Top 5 영상 표
     * 인사이트 3 bullet
   - 색상: YouTube 레드 (#FF0000)

6. 결과 표 출력:
   ✅ 디스코드 발송 완료
   - 메시지 ID: 1234...
   - YouTube API 호출: 3건
   - Discord API 호출: 1건
   - 총 소요: 약 1분
```

수동 작업 비교: YouTube Studio 접속 → 채널 통계 페이지 → 영상별 클릭 → 댓글 일일이 읽기 → 표 정리 → 슬랙 공유 = **약 30분**. MCP 조합으로 **30배 단축**.

## 두 번째 결과물 · 이메일 분석 → 디스코드 자동 분류

**사전 조건**: Gmail MCP (Claude.ai 통합) 사전 활성화. CS 메일 받는 Gmail 계정.

### 명령

디스코드 `#cs-inbox` 채널에서:

```
@봇 지난 24시간 CS 메일 분석해줘

또는 슬래시 명령:
/email-triage 24h
```

### 자동 실행

```
1. Discord MCP · 슬래시 명령 이벤트 수신
   - 이벤트: interactionCreate
   - command: email-triage
   - param: 24h

2. mcp__claude_ai_Gmail__list_messages 호출
   - query: "label:CS newer_than:24h"
   - max_results: 100

3. Claude 가 메일 N개 카테고리 분류:
   - 긴급 (배송 분실·결제 오류·VIP 컴플레인)
   - 환불·교환 (제품 하자·단순 변심)
   - 일반 문의 (사이즈·재입고)
   - 스팸 (광고·외부 영업)

4. 긴급 3건은 본문 일부 발췌 (제목·발신자·요지)

5. mcp__discord__send_message 호출
   - 채널: #cs-inbox
   - embed:
     * 제목: 📧 CS 메일 N건 분류
     * 카테고리 4종 카운트
     * 긴급 3건 미리보기 (제목·발신자·요지)
     * 권장 액션 (환불 8건 일괄 처리 등)
   - 색상: Gmail 레드 (#EA4335)

6. 결과 표 출력:
   ✅ 분류 완료 + 디스코드 발송
   - Gmail API 호출: 1건 (배치)
   - Discord API 호출: 1건
   - 분류 정확도: ~92% (긴급 인식)
   - 총 소요: 약 30초
```

수동 작업 비교: Gmail 열기 → 정렬·필터 → 메일 일일이 읽기 → 카테고리 분류 → 표 만들기 → 슬랙 공유 = **약 20분**. MCP 조합으로 **40배 단축**.

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `Invalid Bot Token` | Token 만료·오타·재발급됨 | Developer Portal > Bot > Reset Token 재발급 |
| 봇이 오프라인 표시 | mcp-discord 프로세스 미실행 | Claude Code 재시작 (새 세션에서 자동 온라인) |
| `Guild not found` | Guild ID 오타 또는 봇 미초대 | Server Settings > Widget > Server ID 재확인 + OAuth URL 로 재초대 |
| 메시지 보내기 실패 (`Missing Access`) | 채널 권한 부족 | 채널 우클릭 > Edit Channel > Permissions > 봇 역할 권한 추가 |
| 메시지 읽기 안 됨 | MESSAGE CONTENT INTENT 미활성화 | Developer Portal > Bot > Privileged Gateway Intents > MESSAGE CONTENT INTENT ON |
| `mcp__discord__*` 도구 안 보임 | `.mcp.json` 문법 오류 또는 재시작 안 함 | `claude mcp list` 등록 확인 + Claude Code 완전 종료 후 재시작 |
| reaction 자동 처리 안 됨 | "Add Reactions" 권한 없음 또는 PRESENCE INTENT 비활성 | 봇 역할 권한 + Developer Portal Intents 활성화 |
| Embed 색상 안 보임 | 색상값 형식 오류 (HEX 대신 정수 필요) | 0x5AE2A8 형식 또는 정수 변환 (5957288) |
| 메시지 N개 이상 못 가져옴 | Discord API limit 100개/요청 | 페이지네이션 (before/after 파라미터) · 자동 분할 |
