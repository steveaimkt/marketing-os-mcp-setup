## 🎯 본인 데이터로 따라하기 (10분)

1. `discord.com/developers/applications` → New Application → Bot
2. Bot Token 발급 → `.env` 에 `DISCORD_BOT_TOKEN=MTxxx` 등록
3. Bot 권한 설정: Send Messages, Embed Links, Read Messages
4. OAuth URL 생성 → 본인 서버에 Bot 초대
5. Claude Code:
   ```
   Discord #marketing 채널에 "안녕하세요" 라고 보내줘
   ```
6. Discord 채널 즉시 확인

## 🔁 marketing-os 전체 자동화의 종점

Discord MCP 는 다른 모든 MCP 의 **결과물 종착지**:

```
GA4 MCP → 리포트 생성 → Discord 발송
Sheets MCP → 분석 완료 → Discord 발송
Ads MCP → 이상치 감지 → Discord 즉시 경고
Buffer MCP → 발행 완료 → Discord 확인 요청
Firecrawl MCP → 경쟁사 변화 → Discord 알림
```

→ 마케터는 Discord 만 보면 모든 자동화 상태 한눈에 파악.

## 💡 자주 묻는 질문

**Q. Webhook vs Bot Token 차이?**  
A. Webhook: 1개 채널만, 간단. Bot: 모든 채널 + 반응 수집·DM 가능. marketing-os 는 Bot 권장.

**Q. 슬랙도 동시 사용 가능?**  
A. 별도 Slack MCP 필요. Claude.ai 통합으로 자동 OAuth 가능.

**Q. 한국어 메시지 깨짐 없이?**  
A. UTF-8 OK. 한국어 임베드 정상 표시.

**Q. 파일·이미지 첨부?**  
A. attach_file 도구로 가능. URL or 로컬 파일 전달.

**Q. 멘션 (@user, @everyone)?**  
A. `<@USER_ID>` 형식으로 자동 처리. allowed_mentions 설정으로 spam 방지.
